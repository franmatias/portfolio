App del tiempo

Desarrollo Web UX/UI Framework

Contexto y Desafío

Este proyecto fue concebido como una propuesta final de curso, integrando todos los conocimientos adquiridos a lo largo del año. El objetivo principal era poner en práctica habilidades de análisis y diseño, enfrentando un reto real que permitiera a los alumnos aplicar metodologías profesionales en el desarrollo de una aplicación web.

Entre los desafíos más destacados se encuentran el trabajo en equipo, el uso de herramientas colaborativas, la documentación y validación de cada fase del proyecto, así como el despliegue y difusión de la aplicación. Todo ello contribuyó a una experiencia de aprendizaje integral y orientada a la realidad profesional.

El proyecto se basa en la creación de una aplicación web que permite a los usuarios consultar el pronóstico del tiempo de manera sencilla y accesible. La plataforma está diseñada para ser intuitiva y fácil de usar, con un enfoque en la experiencia del usuario y la accesibilidad.

El objetivo es proporcionar a los usuarios una herramienta útil y eficiente para consultar el clima, con un diseño atractivo y funcional. La aplicación se basa en la API de OpenWeatherMap, que proporciona datos meteorológicos en tiempo real y pronósticos a corto plazo.

El proyecto se desarrolló en un entorno colaborativo, utilizando herramientas como GitHub para el control de versiones y la gestión del código fuente. Se aplicaron metodologías ágiles para asegurar una entrega continua y un feedback constante entre los miembros del equipo.

Mi Rol

En este proyecto, desempeñé el rol de desarrollador frontend y diseñador UX/UI. Mi responsabilidad principal fue la creación de la interfaz de usuario, asegurando que fuera intuitiva y accesible para todos los usuarios.

Colaboré estrechamente con el equipo de desarrollo backend (alumnos) para integrar la API de OpenWeatherMap y asegurar que los datos se mostraran correctamente en la aplicación. También participé en la investigación y análisis de las necesidades de los usuarios, así como en la creación de wireframes y prototipos en Figma.

Además, me encargué de la documentación del proceso de desarrollo, asegurando que cada fase estuviera bien documentada y que se siguieran las mejores prácticas de desarrollo web. Esto incluyó la creación de guías de estilo, documentación técnica y pruebas de usabilidad.

Mi objetivo fue asegurar que la aplicación no solo cumpliera con los requisitos funcionales, sino que también proporcionara una experiencia de usuario excepcional. Trabajé en la optimización del rendimiento y la accesibilidad de la aplicación, aplicando principios de diseño inclusivo y mejores prácticas de desarrollo web.

El resultado final es una aplicación web que no solo es funcional y eficiente, sino que también es atractiva y fácil de usar. Estoy orgulloso de haber contribuido a este proyecto y de haber trabajado con un equipo tan talentoso y comprometido.

El proyecto fue un gran éxito, y la aplicación ha recibido comentarios positivos de los usuarios. Estoy emocionado de ver cómo evoluciona en el futuro y de seguir contribuyendo a su desarrollo.

Proceso y Metodología

El proceso de desarrollo de la aplicación se llevó a cabo en varias fases, siguiendo una metodología ágil que permitió una entrega continua y un feedback constante entre los miembros del equipo. A continuación, se describen las principales fases del proyecto:

  1. Investigación y Análisis: Se realizó un análisis exhaustivo de webs y aplicaciones de pronóstico del tiempo. Se buscó APIs OpenSource.
  2. Diseño y Prototipado: Se crearon wireframes y mockups en Figma para visualizar la estructura y el flujo de usuario de la plataforma.
  3. Desarrollo e Implementación: El desarrollo se realizó utilizando Visual Studio Code como editor principal y Git para el control de versiones, asegurando un flujo de trabajo colaborativo y seguro.
  4. Pruebas y Lanzamiento: Se realizaron pruebas exhaustivas de la plataforma, incluyendo pruebas unitarias, de integración y de usuario.
  5. Documentación: Se documentó todo el proceso de desarrollo, incluyendo la investigación, el diseño, el desarrollo y las pruebas. Esto permitió una mejor comprensión del proyecto y facilitó futuras mejoras y actualizaciones.
  6. Despliegue: La aplicación se desplegó en Netlify, asegurando que estuviera disponible para su uso y que los usuarios pudieran beneficiarse de sus funcionalidades.
  7. Feedback y Mejora Continua: Se recogió feedback de los usuarios y se realizaron mejoras continuas en la aplicación, asegurando que cumpliera con las expectativas y necesidades de los usuarios.

Este enfoque permitió una colaboración efectiva entre los miembros del equipo y garantizó que la aplicación cumpliera con los estándares de calidad y accesibilidad. A continuación, se detallan las principales fases del proceso:

1

Investigación y Análisis

Se realizó un análisis exhaustivo de webs y aplicaciones de pronóstico del tiempo. Se buscó APIs OpenSource.

  • Comparativa de funcionalidades entre plataformas líderes del sector.
  • Identificación de necesidades de los usuarios a través de encuestas y entrevistas.
  • Evaluación de la accesibilidad y usabilidad de soluciones existentes.
  • Selección de la API OpenWeatherMap por su documentación clara y facilidad de integración.

Este proceso permitió definir los requisitos clave y establecer una base sólida para el diseño y desarrollo de la plataforma.

2

Diseño y Prototipado

Se crearon wireframes y mockups en Figma para visualizar la estructura y el flujo de usuario de la plataforma.

  • Diseño de prototipos de baja y alta fidelidad para iterar rápidamente sobre ideas.
  • Definición de la paleta de colores, tipografía y componentes visuales siguiendo principios de accesibilidad.
  • Validación de los prototipos con usuarios reales para recoger feedback y ajustar el diseño.
  • Documentación de los flujos principales y casos de uso para facilitar la comunicación con el equipo de desarrollo.
3

Desarrollo e Implementación

El desarrollo se realizó utilizando Visual Studio Code como editor principal y Git para el control de versiones, asegurando un flujo de trabajo colaborativo y seguro. Se eligió el framework Vue.js por su flexibilidad y facilidad de integración, complementado con plugins como Vue Router para la gestión de rutas y Vuex para el manejo del estado global de la aplicación.

  • Configuración de un entorno de desarrollo moderno con Node.js y npm.
  • Implementación de componentes reutilizables y modulares en Vue.js.
  • Integración de la API de OpenWeatherMap para obtener datos meteorológicos en tiempo real.
  • Despliegue de la aplicación en Netlify para facilitar el acceso y las pruebas.
  • Uso de herramientas como ESLint y Prettier para mantener la calidad y consistencia del código.

Durante esta fase, se priorizó la accesibilidad, el rendimiento y la experiencia de usuario, aplicando buenas prácticas de desarrollo web y asegurando compatibilidad multiplataforma.

4

Pruebas y Lanzamiento

Se realizaron pruebas exhaustivas de la plataforma, incluyendo pruebas unitarias, de integración y de usuario. Se utilizó Jest para las pruebas unitarias y Cypress para las pruebas end-to-end, asegurando que cada componente funcionara correctamente y que la experiencia del usuario fuera fluida.

  • Pruebas de usabilidad con usuarios reales para identificar áreas de mejora.
  • Optimización del rendimiento y la carga de la aplicación utilizando herramientas como Lighthouse.
  • Corrección de errores y ajustes basados en el feedback recibido durante las pruebas.
  • Documentación del proceso de pruebas y resultados obtenidos.
  • Preparación de la plataforma para el lanzamiento, asegurando que cumpliera con los estándares de calidad y accesibilidad.

Finalmente, se lanzó la plataforma al público, asegurando que estuviera disponible para su uso y que los usuarios pudieran beneficiarse de sus funcionalidades. El despliegue se realizó utilizando la plataforma Netlify para aplicaciones web.
